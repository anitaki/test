Pinokio & Gepeto

Pinokio is an open-source local AI browser — a 1-click launcher for any AI app. Gepeto is Pinokio's built-in script generator that auto-creates install, run, update, and reset launcher scripts. These appear as /pinokio and /gepeto custom commands in Claude Code when you've set up the integration. They are not native Claude Code built-ins — they live in your ~/.claude/commands/ as Markdown files injected by the Pinokio environment.
/pinokio pinokio Interface with the Pinokio local AI browser — install, launch, and control locally-running AI apps. Pinokio is agent-ready: any AI (Claude, Codex, Gemini…) can control apps with zero config.
/gepeto pinokio Gepeto is Pinokio's app-launcher generator. Give it a GitHub/HuggingFace repo and it instantly generates install + run + update + reset scripts. UV is the default Python package manager.
Pinokio home: ~/pinokio/ — all apps isolated here
Gepeto output: install.js · start.js · reset.js · update.js
Package manager: UV (replaces pip in v3.0+)
Works with: Claude, Cursor, Codex, Gemini, Openclaw

Custom commands custom

Store .md files in ~/.claude/commands/ (global) or .claude/commands/ (project). Use $ARGUMENTS for dynamic args, $1 $2 for positional args. They appear in /help instantly.
/cmd $ARGUMENTSPass a dynamic value to any custom command
/cmd $1 $2Positional args in custom commands
@src/main.jsInclude a file's full content in your prompt context
!git statusRun shell command directly — bypasses conversational mode, saves tokens

Skills — natural language triggers skill

Skills live in ~/.claude/skills/ (global) or .claude/skills/ (project) as Markdown files with a description: frontmatter field. Claude reads the description and decides autonomously when to invoke them — no slash needed. Say "make this quieter" or "overdrive this layout" and the right skill fires.

Quick tips

Ctrl+DEnd session (same as /exit)

/compact earlyCompact before hitting limits, not after — pass custom instructions
